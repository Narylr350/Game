# 斗地主联机项目

## 项目简介

这是一个基于 Java 的控制台多人斗地主项目，当前采用客户端 / 服务端的联机方式运行。项目围绕斗地主对局流程展开，逐步实现三人连接、发牌、叫地主 / 抢地主、出牌与阶段流转等核心能力。

项目以多人协作为前提推进开发，不同成员分别负责对局主逻辑、服务端与客户端通信、以及后续的登录注册和数据库持久化模块。

## 当前已实现功能

- 支持服务端启动后等待 3 个客户端连接
- 支持客户端输入玩家名称并接入对局
- 支持房间创建、洗牌、发牌和底牌生成
- 支持叫地主 / 抢地主流程与地主确认
- 支持出牌阶段的基础流程推进
- 支持部分斗地主牌型识别与合法性校验
- 已编写部分单元测试和流程测试，用于校验规则与阶段流转

## 技术栈

- Java 21

- JUnit 5

- MySQL

说明：

## 运行方式

### 环境要求

- JDK 21
- Maven 工程环境

### 启动顺序

1. 先启动服务端入口 `server.net.Server`
2. 再分别启动 3 个客户端进程
3. 客户端入口可使用：
   - `client.Player1`
   - `client.Player2`
   - `client.Player3`
   - 或直接使用 `client.PlayerClient`
4. 三个客户端连接完成后，在控制台中按提示输入内容进行对局

### 说明

- 默认通信地址为 `127.0.0.1:8888`
- 当前交互方式为控制台输入输出
- 本地未验证 Maven 命令行执行环境，因此 README 不把命令行构建结果写成已验证状态

## 团队协作分工

- 一名成员负责核心对局主逻辑
  - 包括房间模型、阶段推进、叫地主 / 抢地主流程、出牌流程和结算相关逻辑
- 一名成员负责服务端与客户端通信
  - 包括 Socket 连接、玩家接入、消息收发、控制台联机流程
- 一名成员负责下一阶段的账号与数据模块
  - 包括登录注册功能
  - 包括使用 JDBC 连接 MySQL
  - 包括将关键出牌数据和对局相关数据写入数据库
