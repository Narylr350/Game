# 斗地主项目总览

## 最终目标

最终要做成一个完整的控制台斗地主项目，能从三人连接开始，依次完成发牌、抢地主、出牌、结算与重开。

## 开发原则

- 发展方式是按阶段逐步增长功能，不是一次性把完整框架预先搭好。
- 先跑通主流程，再考虑抽象和拆分。
- 先完成当前阶段，再进入下一阶段。
- 当前用不到的功能，不提前建框架。

## 推荐结构

项目推荐按 `client / server / game / rule / util` 分层组织。

- `client` 负责客户端入口和交互。
- `server` 负责网络接入和消息转发。
- `game` 负责房间、流程和对局编排。
- `rule` 负责规则判断。
- `util` 负责牌面、牌堆和通用工具。

## 开发阶段

### 第一阶段：连接并发牌

- 3 个客户端能连接到服务端。
- 服务端能收集玩家名字。
- 服务端能完成发牌并把手牌发给各客户端。

### 第二阶段：抢地主

- 跑通叫地主、抢地主和确定地主。
- 底牌合并到地主手牌中。

### 第三阶段：基础出牌

- 跑通首出、跟牌、压制和过牌。
- 完成最基础的出牌合法性判断。

### 第四阶段：结算与重开

- 完成胜负结算、积分或计分结果展示。
- 支持一局结束后重新开始下一局。

## 结构约定

- 网络层只做连接、收发和转发，不承载规则判断。
- `game` 层负责把连接、发牌、抢地主、出牌和结算串成流程。
- `rule` 层只处理规则，不直接碰网络。
- `util` 层只放公共工具，不放对局流程。
