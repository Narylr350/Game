# 斗地主项目人员分工

## 开发者 A：核心逻辑

主负责目录：

- `src/main/java/server/model`
- `src/main/java/server/service`

当前负责内容：

- `GameSession`
- `DealResult`
- `PlayerHand`
- `PlayerState`
- `GameService`
- `GameSessionManager`

工作流程：

1. 先定义或调整模型字段
2. 再补 `GameService` 的流程方法
3. 保证 service 层输出结构稳定
4. 给网络层提供尽量稳定的调用接口
5. 不直接碰 socket 和客户端收发

当前任务：

- 在 `GameSession` 中加入抢地主阶段需要的状态
- 在 `GameService` 中增加地主设置和底牌合并方法
- 设计 `PlayResult` 之类的结果对象
- 让 `GameSessionManager` 承担更完整的对局生命周期

## 开发者 B：网络层

主负责目录：

- `src/main/java/server/net`
- `src/main/java/client`

当前负责内容：

- `Server`
- `PlayerConnection`
- `GameClient`
- `Player1`
- `Player2`
- `Player3`

工作流程：

1. 固定客户端首包格式
2. 把客户端输入转成服务层调用
3. 把服务层结果转成客户端消息
4. 保证连接建立、断开和消息转发稳定
5. 不在网络层自己写规则判断

当前任务：

- 统一请求消息格式
- 统一响应消息格式
- 给 `handleClient(...)` 加基础命令分发
- 把聊天消息和游戏命令区分开
- 视情况增加 `ClientHandler` 或消息分发器

## 开发者 C：规则和玩法

主负责目录：

- `src/main/java/server/rule`

当前负责内容：

- `CardType`
- `CardTypeRule`
- `LandlordRule`
- `PlayRuleChecker`

工作流程：

1. 先定义规则输入和输出
2. 再实现基础牌型判断
3. 再实现抢地主和出牌合法性判断
4. 通过 service 层接入，不直接操作网络层
5. 优先写可单测的方法，不依赖 socket

当前任务：

- 完成单张、对子、三张、炸弹、王炸判断
- 明确顺子、连对、飞机的输入规则
- 定义“是否能压过上一手”的比较逻辑
- 给 `LandlordRule` 补叫地主轮转规则

## 文件边界

### A 主要改

- `src/main/java/server/model/*`
- `src/main/java/server/service/*`

### B 主要改

- `src/main/java/server/net/*`
- `src/main/java/client/*`

### C 主要改

- `src/main/java/server/rule/*`

共同约定：

- 不随便跨边界大改别人主负责目录
- 公共模型变更前先同步字段设计
- 先定接口，再写实现

## 当前建议开发顺序

### 第一阶段

- A 完善 `GameSession`、`DealResult`、`PlayerState`
- B 稳定 `Server + GameClient`
- C 完成规则接口和基础牌型骨架

### 第二阶段

- A 增加地主和出牌流程对象
- B 增加命令分发和消息协议
- C 接入抢地主和出牌合法性判断

### 第三阶段

- 三人联调
- 跑通：
  - 3 人连接
  - 开局
  - 发牌
  - 抢地主
  - 基础出牌
