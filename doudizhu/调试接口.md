# 斗地主项目调试接口

## 统一入口

### 服务端入口

- 启动类：`server.net.Server`

### 客户端入口

- 启动类：`client.GameClient`

## 当前阶段核心接口

当前阶段由 `game.GameFlow` 提供核心流程接口：

```java
DealResult deal(List<String> playerNames);
GameRoom startRoom(List<String> playerNames);
```

调试约定：

- `deal(...)` 用于当前阶段的发牌结果检查。
- `startRoom(...)` 用于从房间入口串起当前阶段流程。
- 网络层只调用这两个入口，不自己拼发牌流程。

## 工具层接口

当前阶段由 `util.CardUtil` 提供工具能力：

```java
List<Integer> createShuffledDeck();
String cardsToString(Collection<Integer> cardIds);
```

调试约定：

- 牌堆生成和牌面展示都统一走 `util.CardUtil`。
- 其他类不要重复写牌面转换逻辑。

## 联调顺序

1. 先用 `game.GameFlow.startRoom(...)` 跑通房间创建和发牌返回。
2. 再启动 `server.net.Server`，确认它能接住 3 个客户端并收集名字。
3. 再启动 3 个 `client.GameClient`，确认它们都能连接并收到各自手牌。
4. 最后把 `GameFlow.startRoom(...)`、`Server` 和 3 个客户端串成完整主流程。
